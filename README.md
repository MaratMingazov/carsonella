# Carsonella

Kotlin Multiplatform + Compose Multiplatform приложение-симулятор физики и химии. Пользователь видит «вселенную» на Canvas, перетаскивает из палитры частицы, атомы, молекулы и «реакторы» — и наблюдает реакции в реальном времени.

## Цель проекта

1. **Изучить, как устроен мир.** Как из элементарных частиц образуются атомы, из атомов — молекулы, из молекул — более сложные соединения, и как в итоге это приводит к появлению жизни и эволюции.
2. **Воплотить это в виде небольшой игры-стратегии.** С одной стороны эволюция идёт своим чередом — можно просто наблюдать. С другой — игрок может вмешиваться в этот мир (пока не решено как именно: возможно копить ресурсы, строить какие-то объекты) и направлять ход эволюции.

## Визуальный стиль и настроение

Целевая эстетика — **созерцательная, расслабляющая, «медитативный микромир»** в духе *Osmos* и *Eufloria*. Игра, в которую играют не спеша, любуясь происходящим.

- Тёмный градиентный фон, ощущение бесконечного космоса.
- Частицы — полупрозрачные светящиеся сферы с мягким halo (радиальные градиенты, аддитивный blend, лёгкий bloom).
- За движущимися частицами тянется затухающий световой след (trails).
- Звёздные реакции — не «вспышка экшена», а тихий всплеск света.
- Минимум UI на канве, ambient-музыка / тишина, никакого хардкора и спешки.

Опционально (космическая фаза): реалистичные цвета звёзд по температуре и туманности на фоне в духе снимков Hubble/JWST.

## Архитектура домена (`shared/.../chemistry/`)

### Сущности

`Entity<State>` — общий интерфейс. Поведения подмешиваются через делегирование (`behavior/*.kt`):

- `DeathNotifiable` — сообщить миру о смерти
- `NeighborsAware` — получать актуальный список соседей
- `ReactionRequester` — отправлять запрос на реакцию в мир
- `EnvironmentAware` — знать своё окружение
- `LogWritable` — писать в общий лог

Конкретные классы: `SubAtom`, `Atom`, `Molecule`, `Star`, `SpaceModule`, `RecombinationModule`. У каждой синхронный `step()`, который вызывается миром раз за tick: двигает сущность, считает силы, при необходимости шлёт `requestReaction(...)`. Никаких собственных корутин или `delay` — мир — единственное «время».

### Элементы

`Element` (enum) + `Details` — большая таблица элементов с параметрами:

- масса, число электронов/протонов/нейтронов, радиус
- `energyLevels` — энергетические уровни (для фотоионизации и спонтанной эмиссии)
- `ion` — ион, который получится при потере электрона
- `recombinationElement` — что получится при захвате электрона
- `alphaReactionResult` — продукт альфа-захвата (нуклеосинтез в звёздах)
- `energyBondDissociation` / `dissociationElements` — для фотодиссоциации молекул

### Окружение

`IEnvironment` — у каждой частицы есть «среда» (центр, радиус, `TemperatureMode.Space | Star`, дети). Любая частица сама может быть средой для других: звезда «содержит» свои протоны/электроны, и альфа-синтез работает только внутри звезды.

### Реакции (`chemistry/chemical_reaction/`)

`ChemicalReactionResolver` хранит список `ReactionRule` и для каждого запроса выбирает правило с максимальным `weight()` (рандом среди равных по весу). Каждое правило возвращает `ReactionOutcome { consumed, spawn, updateState, description }`.

#### Что моделируется сейчас (физическая цепочка)

```
звезда (StarEmission)
    → нуклеосинтез внутри звезды:
        · pp-I (StarPPChain): p+p→D⁺, D⁺+p→³He²⁺, ³He²⁺+³He²⁺→⁴He²⁺+2p
        · pp-II (StarAlphaReaction + StarPPChain): ³He²⁺+⁴He²⁺→⁷Be⁴⁺, ⁷Be⁴⁺+e⁻→⁷Li³⁺, ⁷Li³⁺+p→2 ⁴He²⁺
        · альфа-захват (StarAlphaReaction): ⁴He→⁸Be→¹²C→¹⁶O→…→⁵⁶Ni →(β⁺)→ ⁵⁶Co →(β⁺)→ ⁵⁶Fe (стабильная вершина железного пика), плюс боковая ¹⁵N+α→¹⁹F
        · CNO-I (StarCNOCycle + BetaPlusDecay): ¹²C+p→¹³N, ¹³N→¹³C+e⁺, ¹³C+p→¹⁴N, ¹⁴N+p→¹⁵O, ¹⁵O→¹⁵N+e⁺, ¹⁵N+p→¹²C+⁴He
        · CNO-II (StarCNOCycle + BetaPlusDecay): утечка ¹⁵N+p→¹⁶O, ¹⁶O+p→¹⁷F, ¹⁷F→¹⁷O+e⁺, ¹⁷O+p→¹⁴N+⁴He
        · CNO-III (StarCNOCycle + BetaPlusDecay): утечка ¹⁷O+p→¹⁸F, ¹⁸F→¹⁸O+e⁺, ¹⁸O+p→¹⁵N+⁴He
        · горение углерода (StarCarbonBurning): ¹²C+¹²C → ²⁰Ne/²³Na/²⁴Mg
        · горение кислорода (StarOxygenBurning): ¹⁶O+¹⁶O → ²⁸Si/³¹P/³¹S
    → излучение в космос (StarEmission выбрасывает случайного живого ребёнка звезды наружу)
    → фотоионизация / рекомбинация (PhotoIonization, RecombinationReaction, SpontaneousEmission)
    → аннигиляция позитронов от β⁺-распада с электронами (Annihilation: e⁻ + e⁺ → 2γ)
    → молекулообразование (AtomPlusAtomToMolecule: H+H→H₂, O+O→O₂, O+H₂→H₂O, …)
    → распад молекул фотонами (PhotoDissociation)
```

Это уже физически довольно правдоподобная база — от субатомных частиц через звёздный нуклеосинтез до простых молекул.

**Заметки к цепочке:**

- **`StarEmission` совмещает две роли** через `if/else` (генерация топлива vs выброс наружу). Можно при желании разнести на `StarFuelGeneration` + `StarOutflow`.
- **Внутренний цикл «атом ↔ возбуждённый атом ↔ ион»**: `PhotoIonization` (поглощение) ↔ `SpontaneousEmission` (сброс) и `PhotoIonization` (отрыв) ↔ `RecombinationReaction`. В консоли логов это видно как постоянное «туда-сюда».

### Чего из реальной физики ещё нет

- **BBN — первичный нуклеосинтез.** Почти весь ⁷Li во Вселенной (~10⁻¹⁰ от H) образовался сразу после Большого Взрыва — это **главный** природный источник, не звёзды. Реализация: отдельное правило `BigBangNucleosynthesis`, стартующее в холодной космической среде до зажигания звёзд. Требует ввести понятие **эпох** симуляции (сейчас всё происходит одновременно) — отдельная большая тема, см. «Стратегические заметки».

- **Spallation: ⁹Be, ¹⁰B, ¹¹B, дополнительный канал ⁶Li.** У бора и ⁹Be **нет** звёздного источника: дыры A=5 и A=8 в карте стабильных ядер звёздная α-цепочка перепрыгивает через тройной α-процесс (3⁴He→¹²C), минуя Li/Be/B. У лития есть обходной путь через pp-II (реализовано), у бора — нет. Хуже того: любой бор, попавший в звезду, мгновенно сгорает в протонном фоне (¹⁰B+p→⁷Be+α при ~5 МК, ¹¹B+p→3α) — поэтому в Солнце Li/Be/B в ~10⁹ раз реже C/N/O. Единственный реальный источник Li/Be/B во Вселенной (Reeves–Fowler–Hoyle, 1970) — **космические лучи**: ГэВ-протоны разбивают межзвёздные C/N/O на лёгкие фрагменты. Минорный канал — ν-процесс в сверхновых (~10% ¹¹B), но это другая эпоха. Реализация: правило `CosmicRaySpallation`, имеет смысл подключать только когда в космосе уже накопились тяжёлые ядра. До того момента **B-элементов в `Entity` не заводить** — не хотим декорации без реакций.

- **Нечётные-Z элементы выше N не имеют пути производства.** Альфа-цепочка идёт только по чётным Z (He→C→O→Ne→Mg→Si→…). N покрыт CNO-циклом, Na/P — побочными каналами `StarCarbonBurning`/`StarOxygenBurning`, но Al, Cl, K и далее — пока нет. Главные пути: s-процесс (медленный захват нейтронов, AGB-звёзды) и r-процесс (быстрый захват, сверхновые/нейтронные слияния) — оба требуют свободных нейтронов как реагента, которых у нас в `Entity` пока нет.

- **CNO-циклы — зонтичный термин** для семейства катализированных горений водорода. Все они имеют один паттерн «4p → ⁴He с катализатором», но катализатор и точки утечки разные. Хотим реализовать все, как сделали с радиоактивным распадом — постепенно, по мере необходимости. Каждый следующий цикл «открывается» через редкий канал утечки из предыдущего (физически — конкурирующая реакция с меньшим сечением, у нас — `chance(...)` в `matches()`).

  **Cold CNO (T ~ 10⁷ К, обычные звёзды):**
  - **CNO-I** (или CN-cycle). Старт — ¹²C; цепочка ¹²C→¹³N→¹³C→¹⁴N→¹⁵O→¹⁵N, замыкание ¹⁵N+p→¹²C+α. ✅ **Реализовано** в `StarCNOCycle` + `BetaPlusDecay`. ¹⁴N+p — bottleneck цикла.
  - **CNO-II** (или NO-cycle). Утечка из CNO-I через ¹⁵N+p→¹⁶O+γ (~0.04% в реальности, у нас сжато до 10%). Цепочка ¹⁵N→¹⁶O→¹⁷F→¹⁷O, замыкание ¹⁷O+p→¹⁴N+α. Даёт стабильный ¹⁷O и короткоживущий ¹⁷F. ✅ **Реализовано** в `StarCNOCycle` (два новых кейса) + `BetaPlusDecay` (¹⁷F→¹⁷O).
  - **CNO-III**. Утечка из CNO-II через ¹⁷O+p→¹⁸F+γ (~5%). Цепочка ¹⁷O→¹⁸F→¹⁸O, замыкание ¹⁸O+p→¹⁵N+α. Даёт стабильные ¹⁸O и короткоживущий ¹⁸F (тот самый, что в ПЭТ-сканерах). ✅ **Реализовано** в `StarCNOCycle` (два новых кейса: branching на ¹⁷O+p и замыкание ¹⁸O+p) + `BetaPlusDecay` (¹⁸F→¹⁸O). Реальный ratio замыкание/утечка ~99%/1% сжат до 90%/10% для играбельности.
  - **CNO-IV**. Утечка из CNO-III через ¹⁸O+p→¹⁹F+γ (~5%). Цепочка ¹⁸O→¹⁹F, замыкание ¹⁹F+p→¹⁶O+α. Даёт **стабильный ¹⁹F прямо как промежуточный изотоп цикла**. Альтернативный к ¹⁵N+α путь к фтору.

  **Hot CNO (HCNO, T ≥ 10⁸ К, новые звёзды и x-ray вспышки на нейтронных звёздах):**
  - **HCNO-I** (β-limited). При высокой T протон-захваты идут быстрее β⁺-распадов: ¹³N успевает захватить второй p (¹³N+p→¹⁴O+γ) до того как распадётся; β⁺-распад ¹⁴O→¹⁴N становится узким горлышком вместо ¹⁴N+p. Требует ввести понятие «горячей» температурной моды.
  - **HCNO-II**. Утечка через ¹⁵O+α→¹⁹Ne+γ — **выход из CNO в неон**, цикл больше не замыкается на C/N/O.
  - **HCNO-III**. Через ¹⁸F+p→¹⁹Ne+γ — тоже выход в неон.

  **Продолжения за CNO** (тот же паттерн «p-catalyzed H-burning», но с более тяжёлыми ядрами; в Солнце почти не работают, активны в массивных звёздах):
  - **NeNa-cycle**: ²⁰Ne→²¹Na→²¹Ne→²²Na→²²Ne→²³Na, замыкание ²³Na+p→²⁰Ne+α.
  - **MgAl-cycle**: ²⁴Mg→²⁵Al→²⁵Mg→²⁶Al→²⁶Mg→²⁷Al, замыкание ²⁷Al+p→²⁴Mg+α.

  **Что реализация даст игроку:** через CNO-II/III появятся ¹⁷F, ¹⁷O, ¹⁸F, ¹⁸O; через CNO-IV — стабильный ¹⁹F (естественный путь к фтору). HCNO открывает связь между CNO и α-цепочкой через ¹⁹Ne. NeNa/MgAl — нечётные Z (Na, Al), которые сейчас в проекте появляются только как побочные продукты горения C/O. ²⁶Al — известный космогенный радионуклид, используется как часы для измерения возраста метеоритов.

- **α-индуцированные реакции — зонтичный термин** для всех процессов, где ⁴He входит в столкновение с ядром. По характеру вылетающей вторичной частицы различают каналы (α,γ), (α,p), (α,n), (α,α). Сейчас реализован только первый — будем расширять по мере появления target-ядер.

  - **(α,γ) — чистый α-захват.** ✅ **Реализовано** через `StarAlphaReaction` + поле `Details.alphaReactionResult`. Работает только в `TemperatureMode.Star`. Главная α-цепочка по чётным Z (⁴He→⁸Be→¹²C→¹⁶O→…→⁵⁶Ni) плюс боковые ³He+α→⁷Be (старт pp-II) и ¹⁵N+α→¹⁹F (один из путей к фтору).

  - **(α,p) — α-протонная реакция** (она же реакция Резерфорда). Историческая ¹⁴N+α→¹⁷O+p — первая искусственная ядерная трансмутация (Резерфорд, 1919). При H/He-горении эндотермические каналы (как ¹⁴N, Q = −1.2 МэВ) физически закрыты — тепловые α имеют ~10 кэВ против ~МэВ порога. Открываются: (1) на поздних стадиях горения звёзд (C/O/Si-burning, T ~ 10⁹ К); (2) во **взрывном нуклеосинтезе** — αp-процесс в x-ray вспышках на нейтронных звёздах, каскад (α,p)(p,γ) от ²²Ne до железной группы; (3) на космических лучах в межзвёздной среде; (4) в лаборатории при работе с радиоактивными α-источниками (как у Резерфорда). **Реализация:** новое поле `alphaProtonResult: Element?` в Details + новое правило `AlphaProtonReaction` только в `TemperatureMode.Space` (где α из звёздного outflow встречает холодные ядра — наш аналог «лабораторного» режима). В `TemperatureMode.Star` пока **не** включаем — у нас одна star-мода без различения H/He vs C/O/Si режимов; разрешать (α,p) при любой звёздной T → over-fire против (α,γ). Включим в STAR, когда заведём `TemperatureMode.HotStar` (нужный также для Hot CNO). Конкретные реакции, добавляемые по мере появления элементов:
    - **¹⁴N(α,p)¹⁷O** — Q = −1.2 МэВ. Историческая. Есть всё необходимое — будет первой добавленной.
    - **¹⁹F(α,p)²²Ne** — Q = +1.7 МэВ (экзо). Нужен ²²Ne.
    - **²³Na(α,p)²⁶Mg** — Q = +1.8 МэВ (экзо). Нужен ²⁶Mg.
    - **²⁷Al(α,p)³⁰Si** — нужны Al и ³⁰Si.
    - **³⁵Cl(α,p)³⁸Ar** — нужны Cl и ³⁸Ar.
    - Дальше по αp-процессу — через ⁴²Ca, ⁴⁶Ti, ⁵⁰Cr… до железной группы.

    **Замечание о реагенте-α:** правило должно принимать **любую ⁴He-форму** (²⁺ / ¹⁺ / нейтральный), потому что электронная оболочка для ядерных реакций малозначима — атомный радиус в ~10⁵ раз больше ядерного, эффект «электронного экранирования» порядка 1–2%. Сейчас гелий в проекте есть только как `HELIUM_4_ION_2`, и `StarAlphaReaction` требует именно его — но `AlphaProtonReaction` сразу делаем permissive (принимает ⁴He²⁺ / ⁴He¹⁺ / нейтральный ⁴He). Это закладка на будущее, когда введём рекомбинацию гелия. По мере того, как рекомбинация появится, `StarAlphaReaction` тоже стоит переключить на permissive по тому же принципу.

  - **(α,n) — α-нейтронные реакции.** Главный астрофизический контекст — **источник нейтронов** для s-процесса в AGB-звёздах (без них s-процесс не запустится). Историческая роль не меньше:
    - **1932 — Чедвик открывает нейтрон**, бомбардируя ⁹Be α-частицами от полония и наблюдая загадочное нейтральное излучение. Реакция: ⁹Be + α → ¹²C + n.
    - **1934 — Фредерик и Ирэн Жолио-Кюри открывают искусственную радиоактивность**, бомбардируя ²⁷Al α-частицами: ²⁷Al + α → ³⁰P + n, после чего ³⁰P → ³⁰Si + e⁺ (β⁺, t½ ≈ 2.5 мин). Первое рукотворное радиоактивное ядро — Нобель 1935. Они же делали ¹⁰B(α,n)¹³N и ²⁴Mg(α,n)²⁷Si — те же (α,n) на другие target'ы.

    **Сценарий «искусственная радиоактивность» возникает в нашем фреймворке автоматически**: (α,n) спавнит протон-избыточный продукт, а уже работающий `BetaPlusDecay` через `betaPlusDecayResult` сразу подхватывает его и распадает с эмиссией позитрона. Позитрон встречается с электроном через `Annihilation` → 2γ. Игрок видит на UI всю цепочку Нобеля 1935 без какого-либо нового кода кроме самого (α,n).

    **Реализация** ждёт два пререкизита: (а) **нейтрон как `SubAtom`** — новая частица первого класса, главный блокер; (б) target-ядра. Конкретные реакции по мере появления элементов:
    - **¹³C(α,n)¹⁶O** — главный нейтронный источник для s-процесса в AGB-звёздах. Q = +2.2 МэВ. **¹³C уже есть** (CNO-I), ждёт только нейтрон. **Самая горячая первая цель** — никаких новых Element-ов кроме нейтрона.
    - **¹⁰B(α,n)¹³N** — Жолио-Кюри (один из их target-каналов). Q = +1.1 МэВ. **¹³N уже есть** и **уже распадается** в ¹³C через `BetaPlusDecay`. Нужны только ¹⁰B + нейтрон, и получится готовая историческая демонстрация «искусственной радиоактивности».
    - **⁹Be(α,n)¹²C** — Чедвиковская, Q = +5.7 МэВ. ¹²C **уже есть** (центр α-цепочки). Нужны ⁹Be + нейтрон.
    - **²²Ne(α,n)²⁵Mg** — второй главный источник нейтронов для s-процесса (в массивных звёздах). Нужны ²²Ne, ²⁵Mg, нейтрон.
    - **²⁷Al(α,n)³⁰P** — главный канал Жолио-Кюри. Нужны Al, ³⁰P, ³⁰Si (для замыкания β⁺-распадом), нейтрон.

    **Заметка о боре.** В Spallation-umbrella есть правило «B-элементов в Entity не заводить — не хотим декорации без реакций». Появление ¹⁰B(α,n)¹³N снимает это ограничение **только для ¹⁰B**: у него возникает звёздно-доступная реакция-потребитель. ⁹Be и ¹¹B по-прежнему ждут spallation от космических лучей.

- **⁵⁶Ni → ⁵⁶Co → ⁵⁶Fe — α-цепочка закрыта на долине стабильности.** ✅ **Реализовано** через двойной β⁺-каскад. ⁵⁶Ni (вершина «железного пика», конец α-захвата по чётным Z) больше не «вечен» — он распадается в ⁵⁶Co (t½ ≈ 6 дней в реальности), тот — в стабильный ⁵⁶Fe (t½ ≈ 77 дней). Именно этот каскад — **главный источник видимого света сверхновых типа Ia**: то, что светится в небе через недели после взрыва, — распад ⁵⁶Ni и ⁵⁶Co в выброшенной оболочке. Стандартизированные свечи космологии (Нобель 2011, ускоренное расширение Вселенной) работают только благодаря этой цепочке. Реализация: `betaPlusDecayResult = COBALT_56_ION_27` на ⁵⁶Ni, `= IRON_56_ION_26` на ⁵⁶Co; ⁵⁶Fe — конечная стабильная точка без α-захвата (выше энергетически невыгодно). **Ноль изменений в коде правил** — `BetaPlusDecay` уже generic. ⁵⁶Fe — реальный минимум на кривой энергии связи на нуклон, физически правильный конец нуклеосинтеза в звёздах.

- **Другие нестабильные интермедиаты α-цепочки молчаливо «вечны».** Помимо ⁵⁶Ni (см. выше), у нас есть ещё несколько ядер, которые в реальной природе быстро распадаются, но в симуляции лежат бесконечно — потому что для них не прописан `betaPlusDecayResult`. В звёздной плазме это маскируется тем, что следующий α-захват съедает их раньше, чем они успели распасться. Но как только такое ядро выбрасывается `StarEmission` в космос — там оно «вечное», физически неверно. Полный список из текущих 35 ядер проекта:
  - **⁸Be** — t½ = 10⁻¹⁶ с, **α-распад** обратно в 2⁴He. Особый случай: тройной α-процесс в реальной звезде работает только из-за резонанса Хойла на ¹²C — ⁸Be успевает поймать третью α быстрее, чем сам распасться. У нас динамика грубо аппроксимирована через `chance()` в α-цепочке. Можно оставить, но в `Details` для BERYLLIUM_8_ION_4 поставить комментарий — «физически нестабильно, динамика приближена».
  - **³¹S** — t½ = 2.6 с, **β⁺** → ³¹P. ✅ **Реализовано** через `betaPlusDecayResult = PHOSPHORUS_31_ION_15` на ³¹S. Появляется как продукт `StarOxygenBurning` (¹⁶O+¹⁶O → ³¹S+n канал), теперь корректно распадается в стабильный ³¹P с эмиссией позитрона.
  - **⁴⁴Ti** — t½ = 60 лет, **EC + β⁺** → ⁴⁴Sc → ⁴⁴Ca. Требует завести ⁴⁴Sc и ⁴⁴Ca. ⁴⁴Ti — известный космогенный радионуклид, наблюдается в остатках сверхновых.
  - **⁴⁸Cr** — t½ = 22 ч, **β⁺** → ⁴⁸V → ⁴⁸Ti. Требует ⁴⁸V и ⁴⁸Ti.
  - **⁵²Fe** — t½ = 8 ч, **β⁺** → ⁵²Mn → ⁵²Cr. Требует ⁵²Mn и ⁵²Cr.

  Все фиксы — data-driven через `betaPlusDecayResult` без изменений в правилах. Делать поштучно по мере необходимости: ³¹S→³¹P уже сделан; ⁴⁴Ti, ⁴⁸Cr, ⁵²Fe — когда дойдём до промежуточных Z (Sc, V, Mn) и стабильных аналогов (⁴⁴Ca, ⁴⁸Ti, ⁵²Cr). Заодно это самый дешёвый способ начать заполнять нечётные-Z строки таблицы Менделеева (Sc, V, Mn) — они появятся как промежуточные распадные продукты, без отдельного s-процесса.

- **Радиоактивный распад — зонтичный термин** для нескольких разных ядерных процессов. Часто их путают, но это разные механизмы — будем добавлять по мере появления соответствующих элементов:
  - **α-распад** — ядро выбрасывает ⁴He (Z → Z−2, A → A−4). Тяжёлые ядра: U, Th, Ra, Rn, Po. У нас **обратный** процесс (α-захват) уже есть — `StarAlphaReaction`. Прямой «выброс α» — отдельное правило, понадобится для урановой цепочки (²³⁸U → ²³⁴Th → … → ²⁰⁶Pb, ~14 шагов через чередование α и β⁻).
  - **β⁻-распад** — `n → p + e⁻ + ν̄ₑ` (Z → Z+1, A та же). Нейтрон-избыточные ядра: ¹⁴C, ³H, ⁹⁰Sr, продукты r/s-процесса и деления. У нас сейчас нет нейтрон-избыточных изотопов, правило не нужно. Симметрично β⁺ — то же generic-правило с полем `betaMinusDecayResult` в Details и спавном электрона вместо позитрона.
  - **β⁺-распад** — `p → n + e⁺ + νₑ` (Z → Z−1, A та же). ✅ **Реализовано** как generic `BetaPlusDecay` (data-driven через `Details.betaPlusDecayResult`). Сейчас работает для ¹³N→¹³C, ¹⁵O→¹⁵N (шаги CNO-I), ¹⁷F→¹⁷O (CNO-II), ¹⁸F→¹⁸O (CNO-III).
  - **Electron capture (EC)** — `p + e⁻ → n + νₑ`. Конкурирует с β⁺ при низком Q-value (когда не хватает энергии на пару e⁺/e⁻, порог ~1.022 МэВ). ✅ **Реализовано** как кейс в `StarPPChain` для ⁷Be⁴⁺ + e⁻ → ⁷Li³⁺. Паттерн можно переиспользовать для других протон-избыточных ядер с малым Q.
  - **γ-распад** — возбуждённое ядро снимает энергию мощным γ-квантом, Z и A не меняются. Атомный аналог уже есть — `SpontaneousEmission` (электрон спускается по уровням). Ядерный γ-распад появится, когда введём возбуждённые состояния ядер — пока не моделируем.
  - **Спонтанное деление** — тяжёлое ядро (U, Pu, трансураниды) делится на два больших осколка + несколько нейтронов. Понадобится только если дойдём до моделирования атомной энергии / деления — далёкая перспектива.

  **Универсальная концепция t½** (период полураспада) — статистическое время, за которое половина популяции ядер распадается. Применим к **любому** виду из списка выше. Сейчас в `BetaPlusDecay` и `SpontaneousEmission` темп зашит как `chance(0.02f, ...)` — за тик ~2% шанс, что даёт ~50 тиков (~1 сек) среднего полураспада. Реальные t½ (минуты для ¹³N, миллиарды лет для ²³⁸U) сжаты до зрелищной секунды — сознательное огрубление; точная калибровка по изотопам — отдельный TODO когда наберём 5+ нестабильных видов.

  **Температурная зависимость t½.** В лабораторных условиях (комнатная T) t½ — константа. Но **внутри звезды t½ может резко уменьшаться** из-за того, что распад идёт не только из основного состояния ядра, но и из термически заселённых возбуждённых уровней — а у возбуждённых состояний свои, обычно гораздо более короткие, периоды полураспада. Пример из курса Ишханова (с. 7): ⁹⁹Tc в лаборатории — t½ = 2.1×10⁵ лет, при T > 3×10⁸ К (внутри звезды) — t½ < 10 лет, **ускорение в 10⁴ раз**. У нас сейчас `chance(0.02f)` одинаков в `Star` и `Space` — этот слой физики проигнорирован. Естественная поправка: разный `chance` в зависимости от `getEnvTemperature()`. Не критично пока, но будет важно, если когда-нибудь захотим, чтобы поведение одного и того же нестабильного изотопа отличалось в звезде vs космосе.
- **Нейтринные реакции** — на каждой звёздной реакции в реальности рождаются нейтрино, которые уносят энергию.
- **Химические реакции с переходными состояниями (катализ)** — большинство реальных реакций идут через активационный барьер и могут ускоряться катализаторами.
- **Гравитация** — сейчас выключена (`gravityForce = 0` в `Entity.calculateForce`). Без неё нет планетообразования и аккреции.
- **Кулоновское отталкивание электронных оболочек** — упрощено до простой функции от количества электронов.

Когда дойдём до пребиотической химии и жизни, понадобятся ещё: ферменты, автокаталитические циклы, репликация молекул. Это уже отдельная эпоха симуляции.

## Игровой цикл (`composeApp/.../world/`)

`World` владеет:

- `entities` (`SnapshotStateList<Entity<*>>`)
- палитрой элементов
- корневым `Environment` (радиус 10000)
- `_pendingRequests: MutableList<ReactionRequest>` — буфер запросов реакций
- `random: Random(seed=1L)` — единый сидированный RNG для всей симуляции
- `EntityGenerator`, `ChemicalReactionResolver`

Один tick (`tickMs = 16L`) внутри единственного `_scope.launch`:

1. **Step phase** — `entities.toList().forEach { it.step() }`. Каждая сущность синхронно делает свой ход; может дописать запрос в `_pendingRequests`.
2. **Resolve phase** — берётся снимок `_pendingRequests`, список очищается, каждый запрос проходит через `ChemicalReactionResolver.resolve`, результат применяется (`consumed.destroy()`, `spawn()`, `updateState()`).
3. `delay(tickMs)` — ждём следующий кадр.

Никаких `Mutex`, `Channel` или `suspend` в самой симуляции — tick единственный писатель мира.

## Стратегические заметки (геймплей)

1. **Определить «победу» / прогрессию.**
   Что для игрока — конец партии? Достижение жизни? Технологическое дерево? Колонизация одной звёздной системы? Это направит, какие механики достраивать, а какие — нет.

2. **Разнести симуляцию на эпохи.**
   Между «протон поймал электрон» и «возникла самовоспроизводящаяся молекула» в реальной вселенной 10⁹–10¹⁷ разницы по времени и масштабу — на одной модели всю цепочку не покрыть. Естественные эпохи: космогенез → нуклеосинтез в звёздах → планетарная химия → пребиотика → жизнь. Внутри одной эпохи модель остаётся простой; переходы между эпохами — отдельная игровая веха.

3. **«Реакторы» как игровая механика, обходящая медленные естественные процессы.**
   `SpaceModule` / `RecombinationModule` — уже шаг в эту сторону. Игрок строит «ускорители», которые сжимают невероятно медленные естественные процессы. Это рабочая формула в духе Factorio / Oxygen Not Included, её стоит развивать.
