# Молекулы: граф вместо enum — открытые вопросы

> Миграция **параллельная**: графовые молекулы живут рядом с enum, движок мигрируем по шагам.
> Определения типов — в коде (`shared/.../chemistry/graph/MoleculeGraph.kt`), в доке не дублируем — разъедутся.
> Краткая версия — в [`ROADMAP.md`](../ROADMAP.md). История и уже реализованные решения с обоснованием — в git.

## Текущее состояние (кратко)

Ядро графа (`MoleculeGraph`/`AtomNode`/`Bond`; `mass`/`protons`/`formula`/`canonical`) + `Species = Elemental | Molecular`
(шов `EntityState.element` работает для не-молекул). Химия малых молекул работает:

- **валентность** из Z по октету (`Element.valence()`, CHNO верно; Z>18 → 0);
- **образование** связи (`CovalentBondFormation`, `order=1`), **рост** (`MoleculeGrowth`), **усиление** 1→2→3
  (`BondStrengthening`) — кратные связи эмёрджентны; вода и H₂O₂ собираются. **Рост vs усиление** конкурируют
  в одном `resolve()` (диспетчеризация по инициатору: запросы группируются по `reagents.first().id`) и
  выбираются по энергетическому `weight` (знаковый Q реакции): кислород → `O=O`, углерод → цепи;
- **энергии связей** (`BondEnergy`, эВ по типу связи); образование/усиление излучают фотон. Фотон **улетает быстро**
  (скорость 40) — иначе молекула ловит свой же фотон и получается бесконечный цикл образование↔распад;
- **распад**: `MoleculeGraph.split` (зеркало `merge`) + `PhotoDissociation` (фотон ≥ порога) и `StarDissociation`
  (в звезде, рекурсивно до атомов → далее термоионизация). Порог = `dissociationEnergy` (слабейшая связь, кэш на
  графе), продукты из топологии, электроны гомолитически (нейтральные осколки), избыток энергии на осколках;
- **молекулярная ионизация**: `MolecularPhotoIonization` (фотон `E ≥ IP` → молекулярный катион + `e⁻`; ион ВЫЖИВАЕТ
  как тот же граф с `electrons−1`, смена `Species` не нужна — заряд в динамике; избыток `E − IP` уносит электрон).
  Порог `MoleculeGraph.ionizationEnergy` = минимум атомного IP по графу (кэш, эмёрджентно из состава). Пока
  ДЕТЕРМИНИРОВАНО: `weight=0` → при `E ≥ IP` ионизация бьёт распад, при `порог_распада ≤ E < IP` матчится только распад
  (вероятностное ветвление отложено). Заряд молекулы-иона показывается: `displaySymbol` = `formulaPretty` + суффикс из
  `protons − electrons` (`formulaPretty` остаётся чистой структурой);
- **каноникализация** наивная (перебор, гард ≤ 9 узлов), различает изомеры (этанол ≠ эфир);
- структурный рендер молекулы.

## Открытые вопросы

### Химия малых молекул (Стадия 2)
- **Замыкание колец** — внутримолекулярная связь двух ненасыщенных атомов ОДНОЙ молекулы → кольца/листы/каркасы
  (бензол, графен, алмаз, фуллерены). Нужна геометрия узлов (понять, что концы сблизились) + предпочтение размера
  кольца (5–6 выгодны, 3–4 напряжены). Без него чистый углерод растёт бесконечными цепями; в химии жизни спасает
  терминация водородом (`–CH₃` закрывает слоты).
- **VSEPR-геометрия** — форма молекулы (вода «уголком» 104.5°, а не линией). Простое отталкивание связанных атомов
  даёт формы «без лоун-пар» (2→180°, 3→120°, 4→тетраэдр) — вода вышла бы ЛИНЕЙНОЙ. Изгиб создают неподелённые пары
  → нужна VSEPR-эвристика: угол по стерическому числу (связи + лоун-пары), лоун-пары = валентные − связывающие.
  Требует per-atom электронов (см. ниже).
- **Замещение / метатезис / присоединение** (`A–B + C → A–C + B`) — основа органики (горение `CH₄+O₂`, гидрирование
  `HC≡CH→этан`). Отдельный слой с **гейтом по энергии активации** (иначе всё мгновенно сваливается в глобальный
  минимум — нефизично и комбинаторно взрывается). НЕ встраивать в `MoleculeGrowth` (тот — чистая ассоциация без
  разрыва связей).
- **Гетеролитический распад / ионы-осколки** — сейчас распад только гомолитический (нейтральные радикалы). Для ионов
  делить электроны по электроотрицательности (данные — энергии ионизации по Z).
- Ионы-реагенты; зум/LOD рендера.

### Каноникализация
Наивная (перебор, гард ≤ 9 узлов: 9! ≈ 363k — мгновенно, 10! на порядок дороже) — **сделано**. Открыто: переход на
**Морган-подобный EC** (расширенная связность + разрыв ничьих по типу/заряду/кратностям), когда молекулы перерастут
гард; точная граница. Полимеры каноникализировать не надо — идентичность = последовательность (уже канонична).

### Идентификация: ключ → смысл
Канон отвечает «равны ли молекулы», но смысла не несёт. Лестница: формула (бесплатно) → канон (точная идентичность,
изомеры) → **классификация по функциональным группам** (`–OH`→спирт, `–COOH`→кислота, пептидная связь→белок; тот же
подграф-матчинг, что и реакции — один движок) → **реестр имён**. Открыто: когда вводить классификацию — сразу или
после реестра точных имён?

### Реестр известных молекул (опционально, НЕ ядро)
Курируемая таблица `канонический ключ → {имя, символ, поведение}`. Симуляция работает и без него — молекулы анонимны,
опознаются структурно (это нормально; большинство молекул безымянны). Ценность: имена курируемого подмножества +
**крючок прогрессии** — первое появление воды/аминокислоты/АТФ = событие-веха. Сидируется той же каноникализацией.

### Per-atom заряд/электроны (отложено — большой слой)
Сейчас **заряд — свойство МОЛЕКУЛЫ**, не атома (`MoleculeState.electrons`): при ионизации электрон уходит с
делокализованной орбитали, «какой атом потерял» физически не определено; формальный заряд на атоме — бухгалтерская
условность. Ввести опциональный `AtomNode.charge` + **валентность из электронного баланса** имеет смысл, когда
возьмёмся за то, что без этого НЕ выразить: **озон** (центральный O⁺ с 3 связями), гидроксоний/аммоний, ароматику,
**VSEPR** (лоун-пары). Это меняет и бухгалтерию (сохранение электронов по-атомно, деление при распаде).

### Таксономия Species (выпилить ElementType)
Позже расщепить `Elemental` на `Elementary` (частица) / `Nuclear` (атом), чтобы «вид материи» стал type-safe
(исчерпывающий `when`), а тег `Details.type` ушёл. Классификация `Element → вид` переедет в фабрику `Species.of()`.
Star/модули — контейнеры среды (`IEnvironment`), не материя → в таксономию не входят, `ElementType` уйдёт не разом.

### Стадия 3 — полимеры / жизнь (отдельная эпоха)
Идентичность полимера = **последовательность** (`ATGC…`/аминокислоты; граф-каноникализация не нужна). Репликация/
транскрипция/трансляция — правила высшего уровня. Дальше: компартменты (мембраны), автокаталитические сети и —
фронтир — **эмёрджентная** самокопирующаяся эволюция. NB: репликацию НЕ хардкодить, иначе это анимация жизни, а не
жизнь. Вирусы — не «первая жизнь» (паразиты, нужен хозяин); реалистичный старт — самокопирующийся полимер (RNA-мир)
или протоклетка.

## Что модель осознанно НЕ покрывает
- **Озон O₃** — центральному O нужна валентность 3 (формальный заряд `+`, делокализация ~1.5); фиксированная октетная
  `valence()` + отсутствие per-atom заряда этого не выражают (наивный рост дал бы неверный линейный бирадикал
  `•O–O–O•`). Реальный O₃ собирается `O + O₂` только при подводе энергии, а не эмёрджентно. → курируемый спецслучай;
  починится вместе с per-atom зарядом (см. выше).
- **МО-экзотика** — октет/валентность лишь приближение. Реальный C₂ двойной (наше насыщение даёт тройной),
  парамагнетизм O₂ октет не объясняет. Для химии жизни приемлемо (C₂ не встречается) — МО-теорию не тянем.

## Ориентиры (не зависимости)
- **InChI** — золотой стандарт каноникализации (Морган-подобная нормализация + каноническое мечение; слой связности
  `c…` различает изомеры, `InChIKey` — 27-символьный хеш). C-код, не KMP/Wasm-дружелюбен и избыточен для наших
  крошечных молекул → держим ориентиром. [IUPAC/InChI Trust](https://github.com/IUPAC-InChI/InChI).
- **MOL/SDF (CTfile)** — индустриальный текстовый формат сериализации графа (atom block + bond block + `M CHG`/`M ISO`);
  прямое соответствие `AtomNode`/`Bond`. Потенциальный путь импорта/экспорта и вход для InChI. Наш формат — JSON-DTO.
- **nauty** — общая каноническая форма графа (если наивный перебор и Морган упрутся).
